# ゲーム概要仕様書

## ゲームコンセプト

2Dサイドスクロールアクションゲーム。プレイヤーはステージを進み、ゴールアイテム（Gem）に触れることでクリアとなる。敵に触れると即座にゲームオーバーになるが、敵の上から踏みつけることで倒せる。

## シーン構成

| シーン | 役割 |
|--------|------|
| `StartScene` | ステージ選択画面。解放済みステージのみ選択可能 |
| `MainScene` | ゲームプレイ画面。選択されたステージのプレハブを動的にロードして実行する |

## シーン遷移フロー

```
StartScene
  └─ ステージ選択
       └─ MainScene（ゲームプレイ）
            ├─ クリア → MainScene（次のステージ）or StartScene
            └─ ゲームオーバー → MainScene（同じステージをリトライ）or StartScene
```

## ステージ進行システム

- ステージは Stage1〜Stage6 の6つが存在する（プレハブとして管理）
- `PlayerPrefs` を使って端末にデータを保存し、進行状況を永続化する

| キー | 内容 |
|------|------|
| `TryStage` | 次にプレイするステージのインデックス（0始まり） |
| `ClearStage` | クリア済みの最大ステージインデックス |

## ステージのアンロック条件

- `stageNumber <= ClearStage + 1` を満たすステージのみ選択できる
- つまり「直前のステージをクリアしたステージ」まで解放される

## クリア・ゲームオーバー時の処理

- **クリア時**：クリアBGMを再生し、クリア画面を表示。`Time.timeScale = 0` でゲームを一時停止。`ClearStage` を更新する
- **ゲームオーバー時**：死亡SEを再生し、ゲームオーバー画面を表示。`Time.timeScale = 0` で一時停止
- 一時停止中も `deathSE`・`goalSE` は `ignoreListenerPause = true` で正常に再生される
